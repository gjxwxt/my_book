## 一、点九图 （2024-9-9）

#### 1、定义

1.点九图是android端独有的切图格式；

2.我们需要以1px的纯黑线段来定义图片的可拉伸区域和文本区域；

3.我们要尽可能得压缩可拉伸区域的尺寸；

4.最后的切图命名后缀必须是.9.png。

**程序通过识别圆角矩形四周的1px、#000000的黑线**（需要设计画出），**来达到定义拉伸区域和文本区域的目的。**

![](https://cdn.nlark.com/yuque/0/2022/png/28792475/1663227019661-236d4a45-bb04-4d4e-ab20-c98b728c46c9.png)

#### 2、前端通过border-image实现点九图

##### border-image解析

1. border-image是border-image-source || border-image-slice|| border-image-width|| border-image-outset || border-image-repeat 属性的简写
    
2. border-image-source：定义边框图像的路径；url
    

border-image-slice：定义边框图像从什么位置开始分割；（top\right\bottom\left）

border-image-width：定义边框图像的厚度（宽度）；(不写默认为border-width，如果比border-width大会向内挤遮挡内容，比border-width小会在内部挤压)

![](https://cdn.nlark.com/yuque/0/2022/png/28792475/1663231771395-a686ea68-8ea8-4738-9847-52370a02d05d.png)border-width 是30px，border-image-width 是15px

![](https://cdn.nlark.com/yuque/0/2022/png/28792475/1663231792944-d3c8272b-c1f1-42d1-8371-bf8cf2fbb6cd.png)border-width 还是30px，border-image-width 是60px

border-image-outset：定义边框图像的外延尺寸（边框图像区域超出边框的量）；

border-image-repeat：定义边框图像的平铺方式（stretch、round、repeat）：round和repeat重复方式的区别在于round会将要重复的内容进行等比例的缩放来实现不重叠的效果，而repeat就是单纯的重复，可能会产生重叠的效果

![](https://cdn.nlark.com/yuque/0/2022/png/28792475/1663227766361-4daf1dc3-f8ec-4ca4-86b8-d5f73b76b491.png)

##### 案例

1. 拉伸图片（可作为大屏背景）
    

![](https://cdn.nlark.com/yuque/0/2022/png/28792475/1663230199868-e1470c81-4db6-463c-8f5a-7d8ec17e80e3.png)

```css
  width: 300px;
  min-height: 300px;
  border: 58px solid; //边框宽度设置和图片四个角宽度一致，可以保证不会被挤压
  border-image: url("./image/btn.webp") 58 fill stretch;// 58是slice fill让中间那部分填充满
```

2. 聊天对话框（保证border的宽以及border-image的宽一致）
    

![](https://cdn.nlark.com/yuque/0/2022/png/28792475/1663230933109-5987800b-d2b0-4fda-ba34-6e3509c4f0d5.png)

```css
 .container{
  height: 220px;//图片本身这么大，防止他被挤压
  padding: 0px 50px;//图片左右两边是有宽度的，要想把字放到合适的位置需要padding
  width: fit-content;
  border-width: 1px;
  border-style: solid;
  border-image: url('./image/test2.webp') 0 170 0 170 fill/0px 170px stretch; 
	 //只管左右拉伸，所以上下slice为0，0px 170px是border-image-width，为了保证图片的不被挤压
	 //设置成图片自身的宽度
 } 
.span{
	position: relative;//让字落入其中
  top: 96px;
}
```

## 二、正向与反向代理

	举个正向代理的例子，我(客户端)没有绿码出不了门，但是朋友(代理)有，我(客户端)让朋友(代理)去超市买瓶水，而对于超市(服务器)来讲，他们感知不到我(客户端)的存在，这就是正向代理。  

	举个反向代理例子，我(客户端)去朋友(代理)那买瓶水，并没有说去哪里买，反正朋友(代理)买回来了，对于我(客户端)来讲，我(客户端)感知不到超市(服务器)的存在，这就是反向代理。  
	
	总结：代理客户端为正向代理（科学谷歌上网：服务器不知道是哪个客户端，只要是代理服务器发来的请求就返回数据）（因特网），代理服务器为反向代理（局域网）  
  
	一个任务多个服务器，构成集群。订单和商品人多，用多个计算机节点部署同样的业务，就是订单、商品服务集群。即使某个节点宕机，也不影响服务的提供，必须保障内网互通，通过反向代理可以实现集群和负载均衡，发布静态服务
